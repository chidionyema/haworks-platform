## CRITICAL: Webhook Security Vulnerability
- **File**: src/Shipping/Shipping.Api/Controllers/ShipmentsController.cs:102
- **Issue**: Webhook endpoint has no signature validation. Anyone can call `/webhooks/easypost` and fake tracking updates
- **Impact**: Attackers can mark shipments as delivered when they weren't, trigger false business events, manipulate order status
- **Fix**: Add EasyPost webhook signature validation using HMAC-SHA256 with webhook secret

- **Impact**: If database save fails, event is still published, causing permanent data inconsistency across services
