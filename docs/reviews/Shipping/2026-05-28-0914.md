## CRITICAL: Event Publishing Before Database Commit
- **File**: src/Shipping/Shipping.Api/Controllers/ShipmentsController.cs:69
- **Issue**: `publisher.Publish(new ShipmentCreatedEvent)` is called BEFORE `db.SaveChangesAsync(ct)` 
- **Impact**: Events can be published even if the database transaction fails, causing downstream consumers to process phantom shipments. This violates the transactional outbox pattern and can lead to data inconsistency across services.
- **Fix**: Move `await publisher.Publish()` call to after `await db.SaveChangesAsync(ct)` or use MassTransit's EntityFramework Outbox which automatically ensures atomic publishing.

## CRITICAL: Webhook Security Vulnerability
- **File**: src/Shipping/Shipping.Api/Controllers/ShipmentsController.cs:79
- **Issue**: `[AllowAnonymous]` webhook endpoint with no signature verification or authentication
- **Impact**: Anyone can send fake webhook payloads to manipulate shipment statuses, potentially marking shipments as delivered when they aren't, leading to fraud and customer disputes.
- **Fix**: Implement EasyPost webhook signature verification using their HMAC-SHA256 signature in the `X-EasyPost-Signature` header.
