## CRITICAL: Export Job Authorization Bypass

### CRITICAL: Unauthorized access to any export job
- **File**: src/Audit/Audit.Api/Controllers/AuditExportController.cs:28
- **Issue**: `GetExportStatus` endpoint allows any user with "audit-reader" role to access any job by GUID. No ownership validation.
- **Impact**: Users can access sensitive audit exports they didn't request by guessing/enumerating job IDs
- **Fix**: Add ownership check:
```csharp
public async Task<IActionResult> GetExportStatus(Guid jobId, CancellationToken ct)
{
    var snapshot = await _exportService.GetStatusAsync(jobId, ct);
    if (snapshot == null) return NotFound();
    
    var currentUser = User.Identity?.Name ?? "unknown";
    if (snapshot.RequestedBy != currentUser && !User.IsInRole("audit-admin"))
        return Forbid();
        
    return Ok(snapshot);
}
```
