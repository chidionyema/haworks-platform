### [HIGH]: Cursor parameter injection risk
- **File**: src/Audit/Audit.Infrastructure/Queries/AuditQueryService.cs:61-65
- **Issue**: Base64 decoding of cursor with no validation could crash on malformed input
- **Impact**: Service denial through malformed cursor values
- **Fix**: Wrap decoding in try-catch:
```csharp
private static (DateTimeOffset, Guid) DecodeCursor(string cursor)
{
    try
    {
        var decoded = Encoding.UTF8.GetString(Convert.FromBase64String(cursor));
        var parts = decoded.Split('|');
        if (parts.Length != 2) throw new FormatException();
        return (DateTimeOffset.Parse(parts[0], CultureInfo.InvariantCulture), Guid.Parse(parts[1]));
    }
    catch (Exception ex)
    {
        throw new ArgumentException("Invalid cursor format", nameof(cursor), ex);
    }
}
```
