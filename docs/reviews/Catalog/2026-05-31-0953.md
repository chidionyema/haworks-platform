## CRITICAL: Currency Migration Data Loss Risk

### CRITICAL: Potential data loss in UnitPrice migration
- **File**: src/Catalog/Catalog.Infrastructure/Migrations/20260526120000_UnitPriceCents.cs:11
- **Issue**: The migration converts `decimal` UnitPrice to `bigint` UnitPriceCents using `ROUND("UnitPrice" * 100)::bigint`, which will truncate fractional cents
- **Impact**: If existing products have prices like $19.995, this becomes $19.99 instead of $20.00, causing permanent data loss
- **Fix**: Add validation before migration to check for fractional cents, or use proper banker's rounding
