### CRITICAL: Misleading Success Response on Failure
- **File**: src/Catalog/Catalog.Api/Controllers/CategoriesController.cs:25
- **Issue**: When `CreateCategoryCommand` fails, the code returns `Guid.NewGuid()` to the `CreatedActionResult`, suggesting a resource was created when it wasn't
- **Impact**: Client receives 201 Created with a fake ID when operation actually failed, breaking idempotency and causing data inconsistency
- **Fix**: 
```csharp
var result = await mediator.Send(command, ct);
if (result.IsFailure) return result.ToActionResult();
return CreatedAtAction(nameof(List), new { id = result.Value }, result.Value);
```
