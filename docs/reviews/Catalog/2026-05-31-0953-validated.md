## CRITICAL: Misleading API responses on failure
- **File**: src/Catalog/Catalog.Api/Controllers/CategoriesController.cs:27
- **Issue**: Returns `Guid.NewGuid()` for the resource ID when creation fails: `new { id = result.IsSuccess ? result.Value : Guid.NewGuid() }`
- **Impact**: Clients receive a fake ID that doesn't correspond to any real resource, breaking API contracts
- **Fix**: Don't include an ID in error responses, return proper error details instead
- **Confidence**: 10/10

- **File**: src/Catalog/Catalog.Api/Controllers/ReservationsController.cs:46
