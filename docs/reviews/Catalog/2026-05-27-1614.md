## CRITICAL: Logic Error in API Controller Response Generation
- **File**: src/Catalog/Catalog.Api/Controllers/CategoriesController.cs:27, ProductsController.cs:80
- **Issue**: Both controllers generate `Guid.NewGuid()` when creation fails: `result.IsSuccess ? result.Value : Guid.NewGuid()`
- **Impact**: Clients receive fake resource IDs in Location headers on failures, causing application logic errors and potential security issues
- **Fix**: Return appropriate error responses without Location headers on failure: `if (!result.IsSuccess) return result.ToActionResult(); return result.ToCreatedActionResult(...)`

## CRITICAL: Denial of Service Vulnerability in Demo Endpoints
- **File**: src/Catalog/Catalog.Api/Controllers/DemoTestController.cs:117-165
- **Issue**: `StampedeRequest` accepts unbounded `ConcurrentRequests` and `SimulatedDbLatencyMs` values with no validation
- **Impact**: Attackers can exhaust server resources by requesting millions of concurrent operations or extremely long delays
- **Fix**: Add validation: `[Range(1, 100)] int ConcurrentRequests`, `[Range(1, 10000)] int SimulatedDbLatencyMs`

## CRITICAL: Race Condition in Stock Reservation Logic
- **File**: src/Catalog/Catalog.Infrastructure/Repositories/ProductRepository.cs:102-135
- **Issue**: Transaction logic queries stock availability AFTER rollback, creating TOCTOU race condition
- **Impact**: Could lead to inconsistent stock counts and phantom reservations under high concurrency
- **Fix**: Query available stock within the transaction before rollback, or use proper pessimistic locking

## CRITICAL: Transaction Double-Rollback Bug
- **File**: src/Catalog/Catalog.Infrastructure/Repositories/ProductRepository.cs:126-140
- **Issue**: Manual rollback at line 126, then `finally` disposal attempts second rollback
- **Impact**: Could cause connection pool leaks and unpredictable transaction states
- **Fix**: Remove manual rollback and let `await using` handle disposal, or add transaction state checks

## HIGH: Authorization Bypass in Product Reviews
