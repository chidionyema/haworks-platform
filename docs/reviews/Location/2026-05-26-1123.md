## **CRITICAL:** Empty IdempotencyKey Default Value
- **File**: src/Location/Location.Application/Commands/CreateAddressCommand.cs:46
- **Issue**: `IdempotencyKey { get; init; } = string.Empty;` means all commands without explicit keys are treated as the same idempotent operation
- **Impact**: Multiple address creation requests without idempotency keys will incorrectly prevent legitimate addresses from being created, causing data loss
- **Fix**: Remove default or use `required` modifier, and validate non-empty in validator

## **CRITICAL:** Geocoding Logic Creates Invalid Coordinates  
- **File**: src/Location/Location.Application/Commands/CreateAddressCommand.cs:42-50
- **Issue**: When coordinates are missing, `lat = request.Latitude ?? 0; lon = request.Longitude ?? 0` defaults to (0,0). If geocoding fails, address is saved with coordinates (0,0) 
- **Impact**: Failed geocoding results in addresses stored at Gulf of Guinea (0°N 0°E) instead of failing validation
- **Fix**: Initialize as nullable, only set coordinates after successful geocoding, fail if geocoding required but unsuccessful
