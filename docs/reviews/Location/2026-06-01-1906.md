## CRITICAL: Database Constraint Violation Handling Missing
- **File**: src/Location/Location.Application/Commands/CreateAddressCommandHandler.cs:68
- **Issue**: No error handling for unique constraint violations on address creation. When the unique index `IX_Addresses_Unique_Address` is violated, EF Core will throw `DbUpdateException`, which could poison the DbContext if this handler is ever called from a MassTransit consumer context.
- **Impact**: Database context corruption leading to failed subsequent operations in the same transaction scope.
- **Fix**: Add try-catch around SaveChangesAsync and return appropriate business error for duplicate addresses:
```csharp
try 
{
    await dbContext.SaveChangesAsync(cancellationToken);
} 
catch (DbUpdateException ex) when (ex.InnerException?.Message.Contains("IX_Addresses_Unique_Address") == true)
{
    return Result.Failure<Guid>(Error.Conflict("Address.Duplicate", "Address already exists"));
}
```

### HIGH: Authorization Too Permissive
- **File**: src/Location/Location.Api/Controllers/AddressesController.cs:13 & src/Location/Location.Api/Services/LocationHydrationService.cs:14
- **Issue**: Only basic `[Authorize]` attribute without role-based or resource-based authorization. Any authenticated user can create addresses anywhere and search/hydrate any location data.
- **Impact**: Information disclosure vulnerability and potential for data pollution by unauthorized users.
- **Fix**: Implement proper authorization policies:
```csharp
[Authorize(Policy = "LocationRead")]
[Authorize(Policy = "LocationWrite")] // for create operations
```

### HIGH: Unique Constraint Too Restrictive for Real-World Usage
- **File**: src/Location/Location.Infrastructure/Persistence/LocationDbContext.cs:84-86
- **Issue**: Unique constraint on `Street, City, Postcode, Country` prevents legitimate use cases like multiple apartments in the same building or different business units at the same address.
- **Impact**: Legitimate address creation requests will fail, degrading user experience.
- **Fix**: Either remove the constraint or modify it to include additional identifying information like unit/suite number:
```csharp
entity.HasIndex(a => new { a.Street, a.City, a.Postcode, a.Country, a.Unit })
    .IsUnique().HasDatabaseName("IX_Addresses_Unique_Full_Address");
```
