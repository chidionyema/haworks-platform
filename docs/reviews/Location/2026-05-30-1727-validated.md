## CRITICAL: Spatial Query Performance Risk

### **CRITICAL**: Inefficient spatial distance calculation
- **File**: src/Location/Location.Application/QueryHandlers/GetNearbyAddressesQueryHandler.cs:28-39
- **Issue**: Using `.Distance(point)` in LINQ query forces PostGIS to calculate distance for EVERY row before filtering, instead of using spatial index optimization
- **Impact**: Performance will degrade exponentially as address count grows. With 100K+ addresses, queries could take seconds instead of milliseconds
- **Fix**: Replace with PostGIS ST_DWithin for index-optimized spatial filtering:
```csharp
var results = await dbContext.Addresses
    .FromSqlRaw(@"
        SELECT *, ST_Distance(""Coordinates"", ST_GeomFromText('POINT({0} {1})', 4326)) as distance
        FROM location.""Addresses""
        WHERE ST_DWithin(""Coordinates"", ST_GeomFromText('POINT({0} {1})', 4326), {2})
        ORDER BY distance
        LIMIT {3}", 
        request.Lon, request.Lat, request.RadiusMeters, limit)
    .Select(a => new NearbyAddressDto(a.Id, a.Street, a.Postcode, a.Distance))
    .ToListAsync(ct);
```
- **Confidence**: 9/10
