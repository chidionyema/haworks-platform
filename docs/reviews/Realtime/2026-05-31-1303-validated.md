## MEDIUM: Race Condition in Message Deduplication

### **MEDIUM**: Deduplication check outside transaction creates inconsistent state (Confidence: 8/10)
- **File**: src/Realtime/Realtime.Api/Infrastructure/Persistence/RedisInboxService.cs:28-47

- **Impact**: Failed message storage could leave dedup entry, preventing future retries of the same message

## HIGH: JSON Deserialization Without Error Handling

### **HIGH**: Uncaught deserialization exceptions (Confidence: 9/10)
- **File**: src/Realtime/Realtime.Api/Infrastructure/Persistence/RedisInboxService.cs:66, 70
- **Issue**: `JsonSerializer.Deserialize<InboxEnvelope>` and `JsonSerializer.Deserialize<JsonElement>` can throw, causing entire method failure
- **Impact**: Single corrupted message breaks inbox for all users
- **Fix**: Wrap in try-catch and skip corrupted entries:
```csharp
try
{
    var envelope = JsonSerializer.Deserialize<InboxEnvelope>((string)items[i]!);
    if (envelope is not null)
    {
        var data = JsonSerializer.Deserialize<JsonElement>(envelope.DataJson);
        messages.Add(new InboxMessage { MessageId = envelope.MessageId, MessageType = envelope.MessageType, Data = data });
    }
}
catch (JsonException ex)
{
    _logger.LogWarning(ex, "Skipping corrupted inbox message");
    continue;
}
```
