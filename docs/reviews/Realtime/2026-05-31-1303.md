## CRITICAL: Race Condition in Message Deduplication

- **File**: src/Realtime/Realtime.Api/Infrastructure/Persistence/RedisInboxService.cs:26-30
- **Issue**: Deduplication check (`SetAddAsync`) happens outside the transaction that stores the message
- **Impact**: Concurrent requests with same messageId could both pass dedup check and store duplicate messages
- **Fix**: Include dedup check in the transaction or use a Lua script for atomic dedup+store:
```lua

if redis.call('SADD', KEYS[2], ARGV[1]) == 1 then
    redis.call('LPUSH', KEYS[1], ARGV[2])
    -- ... rest of operations
end
```

- **Impact**: Messages permanently lost on delivery failure
