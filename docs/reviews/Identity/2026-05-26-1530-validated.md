## CRITICAL: Timing Attack Vulnerability in Service Authentication
- **File**: src/Identity/Identity.Api/Controllers/AuthenticationController.cs:217
- **Issue**: Service secret comparison uses `string.Equals(serviceSecret, expectedSecret, StringComparison.Ordinal)` which is vulnerable to timing attacks
- **Impact**: Attackers could potentially brute-force service secrets by measuring response times
- **Fix**: Use constant-time comparison like `CryptographicOperations.FixedTimeEquals()` 
- **Confidence**: 10/10

## CRITICAL: Non-Atomic Token Revocation
- **File**: src/Identity/Identity.Application/LogoutCommand.cs:54,72-73
- **Issue**: Token revocation (line 54) and refresh token removal (lines 72-73) are not performed in a single transaction. Code comments at lines 65-71 acknowledge this is a "known limitation"
- **Impact**: Partial failures can leave tokens revoked but refresh tokens active, or vice versa, creating inconsistent authentication state

- **Confidence**: 10/10
