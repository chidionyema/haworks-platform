## CRITICAL: Service Token Endpoint Security Vulnerability
- **File**: src/Identity/Identity.Api/Controllers/AuthenticationController.cs:200
- **Issue**: The `/service-token` endpoint combines `[AllowAnonymous]` with header-based authentication using `X-Service-Secret`. If this secret is compromised, an attacker has unlimited access to generate service tokens.
- **Impact**: Complete service-to-service authentication bypass. Attackers could impersonate any backend service.
- **Fix**: Add rate limiting, IP restrictions, and consider mutual TLS instead of shared secrets:
```csharp

```

### MEDIUM: Weak Redirect URL Validation
- **File**: src/Identity/Identity.Api/Controllers/ExternalAuthenticationController.cs:155
- **Issue**: `IsValidRedirectUrl` validation might allow bypasses through URL encoding or edge cases
- **Impact**: Open redirect vulnerability in OAuth flow
- **Fix**: Strengthen validation and use allowlist:
```csharp
private bool IsValidRedirectUrl(string? url)
{
    if (string.IsNullOrEmpty(url)) return false;
    
    // Decode multiple times to prevent double-encoding attacks
    var decodedUrl = url;
    for (int i = 0; i < 3; i++) {
        var previousUrl = decodedUrl;
        decodedUrl = Uri.UnescapeDataString(decodedUrl);
        if (decodedUrl == previousUrl) break;
    }
    
    // Use strict allowlist validation
    return _securityOptions.AllowedRedirectHosts
        .Any(allowed => decodedUrl.StartsWith($"https://{allowed}/", StringComparison.OrdinalIgnoreCase));
}
```
