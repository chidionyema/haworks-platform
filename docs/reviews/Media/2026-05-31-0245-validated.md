### **MEDIUM**: Pagination Performance Anti-pattern
- **File**: src/Media/Media.Api/Application/ListMedia.cs:45-46
- **Issue**: Uses offset-based pagination (Skip/Take) which becomes increasingly slow with large page numbers
- **Impact**: Poor performance when users navigate to high page numbers in large media libraries
- **Confidence**: 10/10
- **Fix**: Implement cursor-based pagination using CreatedAt timestamp:
```csharp

```
