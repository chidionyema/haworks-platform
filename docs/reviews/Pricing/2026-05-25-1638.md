## CRITICAL: Inconsistent SaveChanges Pattern Breaking Outbox
- **File**: src/Pricing/Pricing.Api/Controllers/PricingController.cs:79
- **Issue**: Controller calls `_logRepo.SaveChangesAsync(ct)` directly after mediator.Send, but handler comments indicate it deliberately doesn't call SaveChanges because it's used from consumers where outbox commits automatically
- **Impact**: Breaks the intended outbox pattern - could cause inconsistent state where HTTP calls commit logs but consumer calls don't, or double-commits in consumer scenarios
- **Fix**: Remove SaveChangesAsync call from controller and ensure handlers are consistent about their commit responsibility
