### CRITICAL: Event publishing outside transaction boundary
- **File**: src/Merchant/Merchant.Application/Merchants/Commands/ApproveMerchant/ApproveMerchantCommand.cs:22-25
- **Issue**: Event is published BEFORE `SaveChangesAsync()`, creating a transaction boundary violation
- **Impact**: If `SaveChangesAsync()` fails, the event will have been published but the merchant won't be activated, creating inconsistent state across the system

