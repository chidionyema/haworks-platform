## CRITICAL: Race Condition in Merchant Creation

### CRITICAL: Check-then-Act Race Condition
- **File**: src/Merchant/Merchant.Application/Merchants/Commands/CreateMerchant/CreateMerchantCommand.cs:38-42
- **Issue**: Uses `AnyAsync` checks followed by `Add()` without proper locking - vulnerable to concurrent requests
- **Impact**: Two users could simultaneously pass the "owner exists" check and both create merchants, violating business rules
- **Fix**: Use pessimistic locking with FOR UPDATE or let database constraints handle it and improve exception handling:
```csharp
// Better approach - let DB constraint handle it, improve exception handling
catch (DbUpdateException ex) when (ex.InnerException is PostgresException { SqlState: "23505" } pgEx)
{
    if (pgEx.ConstraintName?.Contains("IX_Merchants_OwnerId") == true)
        return Result.Failure(Error.Conflict("Merchant.OwnerExists", "Owner already has a merchant"));
    if (pgEx.ConstraintName?.Contains("IX_Merchants_Slug") == true)
        return Result.Failure(Error.Conflict("Merchant.SlugExists", "Slug is already taken"));
    throw;
}
```
