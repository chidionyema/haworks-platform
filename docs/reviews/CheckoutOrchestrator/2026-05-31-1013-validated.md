## CRITICAL: Data Integrity Violation in Invalid Amount Handling
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Application/Sagas/CheckoutSaga.cs:113
- **Issue**: When TotalAmountCents <= 0, the saga sets it to 0 and stores this invalid state in the database
- **Impact**: Creates misleading audit data showing $0.00 checkouts that were actually invalid, breaking financial reconciliation
- **Fix**: Store the original invalid amount with failure reason: `ctx.Saga.TotalAmountCents = ctx.Message.TotalAmountCents; ctx.Saga.FailureReason = $"invalid_amount_{ctx.Message.TotalAmountCents}";`
- **Confidence**: 9/10

## CRITICAL: Race Condition in Saga Creation
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Application/Commands/StartCheckoutCommand.cs:33-36
- **Issue**: Handler explicitly avoids checking for existing sagas to prevent DbContext poisoning, but this creates a race condition where duplicate HTTP requests can create multiple saga instances
