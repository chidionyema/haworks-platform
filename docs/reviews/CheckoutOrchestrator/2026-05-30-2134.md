## ## CRITICAL: Saga Compensation Failure Risk
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Application/Sagas/CheckoutSaga.cs:443-454
- **Issue**: `DeserializeItems` method intentionally throws on corrupt JSON, but this can permanently break stock compensation. If `ReservedItemsJson` gets corrupted in the database, the saga will throw during compensation, leaving stock permanently locked.
- **Impact**: Reserved inventory becomes permanently unavailable, requiring manual database intervention. In a high-volume system, this could lock significant amounts of stock.
- **Fix**: Add fallback logic to log corruption and attempt manual resolution or use a separate audit table to track reservations independently of JSON serialization.

## ## CRITICAL: Race Conditions in Timeout Handling
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Infrastructure/Workers/PaymentExpiryWatcher.cs:100-140
- **Issue**: Both the MassTransit scheduler and the `PaymentExpiryWatcher` can fire timeout events for the same saga. While the saga has some protection via state checks, there's a race window where multiple `PaymentExpiredEvent`s could be processed.
- **Impact**: Potential double-compensation (multiple `StockReleaseRequested` events) or inconsistent saga state transitions.
- **Fix**: Add idempotency tokens to timeout events or use distributed locking to ensure only one timeout source can act on a saga.

## ## CRITICAL: Deterministic GUID Collision Risk
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Application/Commands/StartCheckoutCommand.cs:47-49
- **Issue**: SagaId generation uses only first 16 bytes of SHA256 hash, creating collision risk. With 2^64 possible values and birthday paradox, collisions become probable with millions of checkouts.
- **Impact**: Saga correlation conflicts leading to events being delivered to wrong saga instances, causing incorrect state transitions and potential data corruption.
- **Fix**: Use full SHA256 hash or add additional entropy (timestamp, machine identifier) to reduce collision probability.

- **Issue**: No foreign key constraints between `CheckoutSagaState.OrderId` and actual orders, or between `PaymentId` and actual payments. Saga can reference non-existent entities.
- **Impact**: Data integrity violations, orphaned saga states, and difficulty in data reconciliation during incidents.
- **Fix**: Add foreign key constraints with appropriate cascade behaviors, or implement application-level validation to verify referenced entities exist.
