## HIGH Issues:

### HIGH: Potential Cross-User Saga Interference
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Application/Commands/StartCheckoutCommand.cs:47
- **Issue**: Saga ID is generated only from IdempotencyKey without including UserId, potentially allowing different users with the same idempotency key to interfere with each other's sagas
- **Impact**: If idempotency middleware fails, different users using the same idempotency key could get the same saga ID, leading to data corruption or unauthorized access
- **Fix**: Include UserId in hash input: `var hashInput = $"{request.UserId}:{request.IdempotencyKey}";`

### HIGH: JSON Deserialization Failure During Compensation
- **File**: src/CheckoutOrchestrator/CheckoutOrchestrator.Application/Sagas/CheckoutSaga.cs:451
- **Issue**: `DeserializeItems` throws exceptions during stock release compensation if JSON is corrupted
- **Impact**: Corrupted saga state JSON could permanently lock reserved stock, as compensation fails and stock is never released
- **Fix**: Add error handling in compensation paths:
```csharp
private static IReadOnlyList<StockReservationItem> DeserializeItems(string? json)
{
    if (string.IsNullOrEmpty(json)) return Array.Empty<StockReservationItem>();
    
    try 
    {
        return JsonSerializer.Deserialize<List<StockReservationItem>>(json) 
            ?? Array.Empty<StockReservationItem>();
    }
    catch (JsonException ex)
    {
        // Log corruption and return empty list to allow compensation to proceed
        // Ops can manually investigate and fix via ManualResolutionEvent
        return Array.Empty<StockReservationItem>();
    }
}
```
