## CRITICAL: Unbounded Object Deserialization Security Risk

### **CRITICAL**: AiController accepts unvalidated object parameters
- **File**: src/BffWeb/BffWeb.Api/Controllers/AiController.cs:29, 47, 74, 85
- **Issue**: All endpoints accept `[FromBody] object body` and forward directly to AI service without validation
- **Impact**: Enables JSON deserialization attacks, potential for malicious payloads to crash AI service or leak data
- **Fix**: Define specific DTOs with validation:
```csharp
public sealed record SearchRequest(string Query, int? MaxResults = 10);

[HttpPost("search")]
public async Task<IActionResult> SemanticSearch([FromBody] SearchRequest request, CancellationToken ct = default)
```

### **CRITICAL**: Service token provider masks authentication failures
- **File**: src/BffWeb/BffWeb.Infrastructure/Authentication/BffServiceTokenProvider.cs:52-58
- **Issue**: Catches all exceptions and returns null, masking network failures as "no auth configured"
- **Impact**: Silent authentication bypasses - checkout requests fail with 403 instead of proper service errors
- **Fix**: Distinguish between configuration errors and transient failures:
```csharp
catch (HttpRequestException ex)
{
    _logger.LogError(ex, "Network failure obtaining service token - should retry");
    throw;
}
catch (Exception ex)
{
    _logger.LogError(ex, "Configuration error obtaining service token");
    return null;
}
```
