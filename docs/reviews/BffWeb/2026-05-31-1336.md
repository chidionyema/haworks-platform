## CRITICAL: CheckoutController Idempotency Violation

### CRITICAL: Non-deterministic saga and order IDs break idempotency
- **File**: src/BffWeb/BffWeb.Api/Controllers/CheckoutController.cs:44-45
- **Issue**: `var sagaId = Guid.NewGuid(); var orderId = Guid.NewGuid();` generates different IDs on each call
- **Impact**: Violates idempotency guarantees - retry of same checkout request creates multiple sagas/orders, leading to duplicate charges and inventory corruption
- **Fix**: Derive IDs deterministically from idempotency key: `var sagaId = IdempotencyKey.DeriveGuid(idempotencyKey, "saga"); var orderId = IdempotencyKey.DeriveGuid(idempotencyKey, "order");`
